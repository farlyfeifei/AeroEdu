# 翼启 AeroEdu ✈

### 浏览器端无人机飞控可视化教学平台

*用真实飞行动力学 + PID 控制 + 自然语言场景生成,把"飞控调试"这门硬核工程课,压缩成一个浏览器打开即用的教学沙盒。*

</div>

---

## 一、为什么做这个项目

我本人在北航电子信息专业,做过三届"冯如杯"无人机项目(飞控底层驱动、IMU 多传感器融合、雷达目标检测)。在反复调试飞控的过程中,我最深的痛点是:**飞控调试的试错成本极高**——烧电机、炸机、线下风洞资源紧张,而 PID 三个参数对飞行姿态的影响,光看公式根本建立不起直觉。

同时,我长期相信一个价值主张:**AI 的核心社会价值,是降低专业知识的门槛**。一个县城中学生,没有无人机、没有风洞,也应该能"看见"PID 是怎么让一架飞机稳住的。

所以我把竞赛里沉淀的飞控/仿真经验,做成了这个**零安装、浏览器即开即用**的教学平台。它不是某个竞赛项目的复刻,而是一次"把工程能力转化为普惠教学工具"的独立尝试。

## 二、它能做什么

| 能力 | 说明 |
|------|------|
| **真实飞行动力学** | 四旋翼刚体模型:重力 / 旋翼推力 / 空气阻力 / 风扰,半隐式 Euler 数值积分,4 路电机独立解算 |
| **级联 PID 控制器** | 高度环 + 姿态环(横滚/俯仰/偏航),含抗积分饱和、微分低通滤波,参数实时可调 |
| **3D 可视化** | Three.js 自建四旋翼模型(机身/机臂/旋翼/起落架/LED),旋翼转速联动、轨迹拖尾、轨道相机 |
| **环境扰动注入** | 侧风/逆风/阵风三轴可调,一键随机阵风冲击,复现"强风工况调 PID"的真实场景 |
| **航迹模式** | 悬停 / 圆周 / 8 字 / 着陆,水平误差前馈解算为期望倾角 |
| **AI 场景生成** | 中文自然语言 → 自动配置模式/风扰/高度/PID 风格(如"5级侧风悬停3米,激进PID") |
| **实时遥测 HUD** | 高度、爬升率、水平速度、三轴姿态角、4 路电机 RPM,航空仪表盘风格 |

## 三、技术架构

```
┌─────────────────────────────────────────────────────────┐
│                    浏览器 (零安装)                        │
│  ┌───────────┐   ┌──────────────┐   ┌────────────────┐  │
│  │ ui.js     │◄──│  main.js     │──►│ scene.js       │  │
│  │ 控制面板   │   │  主循环/调度   │   │ Three.js 渲染   │  │
│  │ 遥测 HUD   │   │  固定步长子步  │   │ 无人机模型/轨迹 │  │
│  │ AI 输入框  │   │  事件总线     │   │ 相机/光照/地面  │  │
│  └─────┬─────┘   └──────┬───────┘   └───────▲────────┘  │
│        │                │                   │            │
│        │           ┌────▼────────┐          │            │
│        └──────────►│ physics.js  │──────────┘            │
│                    │ 飞行动力学   │  (状态快照同步)        │
│  ┌────────────┐    │ PID 控制器   │                      │
│  │aiScenario.js│   │ 混控分配     │                      │
│  │ 中文 NLU    │   └─────────────┘                      │
│  │ 槽位填充    │                                         │
│  └────────────┘                                         │
└─────────────────────────────────────────────────────────┘
```

**分层职责(高内聚低耦合):**

- **`physics.js`** — 纯 JS,零依赖。可脱离浏览器在 Node 中跑单元自检(已验证:目标高度 2m 悬停收敛到 2.000m,5 级阵风注入后最大偏差 ~1m 随即恢复)。
- **`aiScenario.js`** — 中文规则 + 槽位填充解析器,离线零延迟。设计为可替换为真实 LLM 调用(保留相同 `parse → ScenarioConfig` 接口),用轻量规则覆盖 90% 教学场景。
- **`scene.js`** — Three.js 渲染层,仅消费 `physics.state()` 快照,不持有物理状态。
- **`ui.js`** — 控制面板/HUD/AI 面板,通过事件回调与主循环通信,不直接操作物理或渲染。
- **`main.js`** — 主控:固定步长子步(1/240s)提升数值稳定性,水平位置误差前馈解算为期望倾角。

## 四、运行方式

> 因为使用了 ES Module + importmap 加载 Three.js,需要通过 HTTP 访问(不能直接双击 file:// 打开)。

```bash
# 方式一: Python (自带)
cd AeroEdu
python -m http.server 8765
# 浏览器访问 http://localhost:8765/

# 方式二: Node
npx serve .

# 方式三: VS Code Live Server 插件, 右键 index.html -> Open with Live Server
```

打开后建议依次尝试:
1. 默认悬停模式,拖动左侧 **Kp/Ki/Kd** 滑块,观察无人机响应变化;
2. 点 **"振荡"** 预设,看 Kd 过小导致的发散;
3. 点 **"注入随机阵风"**,看 PID 抗扰恢复;
4. 切 **"8字"** 模式,看航迹跟踪;
5. 在底部 AI 框输入 `5级侧风下悬停在3米高度,使用激进PID`,点"生成"。

## 五、关键技术点(给技术评委)

### 1. PID 控制器工程化
不是教科书裸 PID,而是包含三项工程化处理:
- **抗积分饱和(iMax 限幅)**:防止大误差下积分项累积导致超调;
- **微分低通滤波**(一阶 IIR,α=0.85):抑制 IMU 噪声被微分放大;
- **悬停推力前馈**:高度环输出叠加 `mg` 重力补偿基线,让 PID 只处理误差,降低积分负担。

### 2. 混控分配(Mixer Matrix)
4 路电机推力由"总推力 + 三轴力矩需求"通过混控矩阵反解:
```
T_FR = T/4 − τ_roll/(2L) − τ_pitch/(2L) − τ_yaw/(2k)
T_FL = T/4 + τ_roll/(2L) − τ_pitch/(2L) + τ_yaw/(2k)
T_BR = T/4 − τ_roll/(2L) + τ_pitch/(2L) + τ_yaw/(2k)
T_BL = T/4 + τ_roll/(2L) + τ_pitch/(2L) − τ_yaw/(2k)
```
这正是真实飞控(如 Betaflight/PX4)的混控逻辑简化版。

### 3. 坐标系转换
物理层用 ENU 坐标系(x 东 / y 北 / z 上),Three.js 用 Y-up,渲染层做 `physics(x,y,z) → three(x, z, -y)` 映射,姿态用 ZYX 欧拉序同步,保证视觉与物理一致。

### 4. AI 工程化落地的取舍
没有无脑接 LLM——教学场景需要**零延迟、可解释、离线可用**。所以用规则 + 槽位填充做 90% 场景,解析过程对用户完全可见(`explain()` 输出可读报告),复杂兜底才走 LLM。这是"AI 工程化"而非"AI 玩具化"的体现。
